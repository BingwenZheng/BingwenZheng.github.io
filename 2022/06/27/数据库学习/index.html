MySQL 数据库

对表进行查询操作：

    show tables

创建表

create table 表名（
字段1 类型 comment ，… ，最后一句不用加逗号）comment

SQL

修改表

  • 修改数据类型

    ALTER TABLE 表名 MODIFY 字段名 新数据类型（长度）

  • 修改字段名和类型

    ALTER TABLE CHANGE 旧字段名 新字段名 类型（长度） [注释][约束]

  • 删除字段

    ALTER TABLE 表名 DROP 字段名;

  • 修改表名

    ALTER TABLE 表名 RENAME TO 新表名

  • 删除表名

    DROP TABLE 表名;

DML(增删改查)

  • 增加数据

    给指定字段添加数据 INSERT INTO 表名 （字段1，字段2，…） VALUES (值1，值2，…)
    给全部字段添加数据 INSERT INTO 表名 VALUES （值1，值2，…）

      insert into employee(id,workno,name,gender,age,idcard,entrydata)   values(1,'1','itcast','male',10,'123455677878','2000-09-09');//选择字段
  insert into user values (2,'xiaohong',18),(3,'xiaoliang',18);//所有的字段批量赋值

  • 修改数据

    update 表名 SET 字段名1 = 值1，字段名2 = 值2，…[where 条件];

      update user set name = 'zzbw' where id = 3;

  • 删除数据

    DELETE FROM 表名 [where 条件]；

      delete from user where id = 2;

  • 查询数据

    基本查询 (select from)–设置别名 as 去重 select distinct from

      select * from user; -- 尽量不用
  select name,workno,age from user;
  -- 设置别名：--select *distinct* id as '编号' from user; -- as可以省略，distinct 去重

条件查询 (where)

    select age from user where age >= 15 and age <= 20;
    select age from user where age between 15 and 20;

    select age from user where id is not null;

    select age from user where gender = 'male' and age < 29;

    select age from user where age = 10 or age = 20;
    select age from user where age in(10,20);

    select age from user where name like '__';-- 查询名字为两个字符
    select age from user where id like '%x'; -- 查询最后一位是x的用户

聚合函数 (count max min avg sum)

    select count(*) from school.subject; -- null值不参与聚合运算

    select avg(classhour) from school.subject;

    select max(classhour)from school.subject;

    select sum(classhour) from school.subject where classhour = 100;

分组查询 (group by)

语法： select 字段列表 FROM 表名 where group by 分组字段名 having 分组后过滤条件
where与having的区别

  • 执行时机不同 ：where是分组之前进行过滤，不满足where不参与分组，having是对分组之后的结果进行过滤。

  • 判断条件不同 ： where不能对聚合函数进行判断，而having可以。

  • 执行顺序不同 ： where > 聚合函数 > having

  • 分组之后，查询的字段一般为聚合函数和分组字段，查询其他字段毫无意义

      select avg(classhour) '课时' from school.subject where gradeid < 8 group by gradeid having avg(classhour) >= 120;
  select gradeid,avg(classhour) '课时' from school.subject where gradeid < 8 group by gradeid having avg(classhour) >= 120;-- 别名最好不要用在后续的筛选条件里

排序查询 (order by)

语法： select 字段列表 from 表名 order by 字段1，排序方式1，字段2，排序方式2…;

排序方式：

  • ASC：升序（默认）

  • DESC：降序

  • 如果是多字段排序，当第一个字段相同时，才会根据第二个字段进行排序。

      select * from school.subject order by classhour desc;

  select * from school.subject order by classhour desc,gradeid desc,subjectname asc;

分页查询 (limit)

语法： select 字段列表 from 表名 limit 起始索引，查询记录数；

注意：

  • 起始索引从0开始，起始索引 = （查询页码-1）*每页显示的页码数

  • 分页查询是数据库的方言，不同的数据库有不同的实现，MySQL是limit

  • 如果查询的是第一页数据，起始索引可以省略，直接简写为limit 10

      select *from school.subject limit 0,10;
  select *from school.subject limit 10,10; -- 此处每页显示10条记录

几个案例：

    select age from user where age in(20,21,22) and gender = 'female';

    select * from user where (age <= 40 and age >= 20) and gender = 'male' and name like '___';

    select gender,count(*) from user where age < 60 group by gender;

    select name,age from user where age <= 35 order by age asc,workdate desc;

    select * from user where gender = 'male' and age between 20 and 40 order by age asc,hourdate asc limit 0,5;

DQL的实行顺序：

from–where–group by–select–order by – limit

约束

  • 非空约束— null
  • 唯一约束— unique 所有字段唯一不重复
  • 主键约束— primary key

表中的任何列都可以作为主键，只要它满足一下条件：

1、任何两行都不具有相同的主键值

2、每个行都必须具有一个主键值（主键列不允许NULL值）

  • 默认约束— default 没有指定则使用默认
  • 检查约束— check(约束条件)
  • 外键约束— foreign key—-日常开发中禁止使用

多表查询

  • 多表关系

  • 多表查询概述（笛卡尔积现象）– 需要消除无效的笛卡尔积

              select * from school.grade,school.student where grade.gradeid = studentno;

  • 多表查询的分类

    连接查询

1.内连接：查询AB交集部分的数据

    select emp.name , dept.name from emp,dept where emp.dept_id = dept.id ;-- 隐式内连接
    select e.name,d.name from emp e,dept d where e.dept_id = d.id; -- 使用别名，方便书写。使用了别名就不要使用表名

    select e.name,d.name from emp e inner join dept d on e.dept_id = d.id; -- 显式外连接

2.外连接：左外：查询左表加交集。右外：查询右表加交集

    select e.*,d.name from emp e left outer join dept d on e.dept_id = d.id; -- 左外连接

    select d.*,e.* from emp e right outer join dept d on e.dept_id = d.id; -- 右外连接

3.自连接：当前表与自身的连接查询，自连接必须使用表别名

    select e.*,d.name from emp e left outer join dept d on e.dept_id = d.id; -- 左外连接

    select d.*,e.* from emp e right outer join dept d on e.dept_id = d.id; -- 右外连接

  • 联合查询 union

      select * from emp where salary < 5000
  union all
  select * from emo where age > 50; -- 将两次查询的结果合并到一起，去掉all关键字可以去重

对于联合查询的多张表列数必须保持一致，字段类型也需要保持一致。

子查询（SQL语句中嵌套SELECT语句，成为嵌套查询）

  • 标量子查询（返回的结果是单个值）

      select * from emp where dept_id = (select id from dept where name = '销售部'); -- 结果只有一个数字。本质就是一个嵌套

  • 列子查询（返回结果为一列可以是多行）

      select * from emp where dept_id in(select id from dept whee name = '销售部' or name = '市场部');

  • 行子查询

      select * from emp where (salary,managerid) = (select salary , manageid from emp where name = 'XXX');-- ()中可以包括不同数据对应相等

  • 表子查询

       select * from where (jpb,salary) in (select job , salary from emp where name = 'XXX' or name = 'YYY');-- 与前面的=做区分，注意in的用法

事务

事务操作(类似于git的实现逻辑)

  • 查看/设置事务的提交方式

      select @@autocommit;-- 值为1则自动提交
  set @@autocommit = 0;

  • 开启事务— start transaction/begin

前两个操作二选一即可

  • 提交事务— commit;
  • 回滚事务— rollback;

事物的四大特性（ACID）

  • 原子性 ： 事务是不可分割的最小操作单元，要么全都成功，要么全部失败
  • 一致性 ： 事务完成时，必须使所有的数据都保持一致状态
  • 隔离性 ： 数据库提供的隔离机制，保证事务在不受外部并发操作影响的独立环境下运行
  • 持久性 ： 事务一旦提交或是回滚，他对数据库的改变就是永久的

并发事务引发的问题

  • 脏读
  • 不可重复读
  • 幻读

事物的隔离级别

    - read uncommitted
    - read committed
    - repeatable read
    - serializable
  • 查看事务隔离级别

select @@transaction_isolation

  • 设置事务的隔离级别

set [session/global] transaction isolation level {四个级别}
