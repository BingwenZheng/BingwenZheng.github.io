发表于2022-06-24|更新于2022-06-30|技术学习

控制的不同版本

  • SVN集中式
    必须联网才能使用，从中央服务器获取最新版本
  • git分布式
    没有中央服务器，在每个人电脑上工作

基本的Linux命令

  • cd..回到上一级
  • cd 进入文件目录
  • pwd 显示当前目录
  • clear 清屏
  • ls 列出当前文件夹下所有文件
  • touch+文件名 生成新的文件
  • rm 移除文件
  • mkdir 创建新的文件夹
  • rm -rf / 删库跑路
  • history 查看命令历史
  • exit 退出

环境配置

Git基本理论

基本命令

  • git add . 添加到暂存区 && git checkout
  • git commit -m 提交信息 文件 && git reset
  • git push && git push
  • Workspace 工作区，存放代码
  • Index 暂存区，本质是文件
  • Repository 仓库区，有所有版本数据，HEAD指向最新放入仓库的版本 master 主分支
  • Remote 远程仓库 Github Gitee 托管代码的服务器

本地搭建仓库

  • git init 初始化
  • git clone url（直接复制网站）

Git文件操作

文件的4种状态

  • git status [filename] 查看文件状态

    忽略文件

  • .git ignore 忽略添加文件

Git版本穿梭（影响工作区域）

git reset –hard 版本号

使用码云（gitee）

  • 创建自己的远程仓库

    IDEA中集成Git

    1.新建项目，绑定git
  • 将远程git文件拷贝到项目中即可

2.修改文件

  • 添加到暂存区
  • commit提交—git log 可以查看提交信息
  • push到远程仓库

3.提交测试

Git分支

列出所有本地分支

  • git branch -v

    远程分支

  • git branch -r

    新建分支，但仍停留在当前

  • git branch [branch-name]

    修改分支

  • 切换分支

  • git checkout 分支名

如果多个分支并行执行，就会导致代码不冲突，存在多个版本

合并分支

  • git merge [branch] （站在当前分支，将别的分支合并进来）

冲突合并 –必须人为决定代码去留

  • 将代码中修改前的代码删掉，将特殊字符删掉即可

创建远程库的别名（方便书写）

  • git remote -v 查看已有别名
  • git remote add 库的地址，创建别名

    推送分支

  • git push 库名 分支名

    拉取分支

  • git pull 库名 分支名
