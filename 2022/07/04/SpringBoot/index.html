创建springboot程序（基于jdk8）

运行springboot要选用application后缀文件

  • parent
  1. 开发springboot继承spring-boot-starter-parent
  2. spring-boot-starter-parent定义了若干依赖管理
  3. 可以避免依赖版本冲突
  4. 可以不用继承，直接引用spring-boot-dependencies依赖也可
  • starter
  1. 减少依赖配置
  • 实际开发
  1. 使用任意坐标时，仅书写GAV中的G和A，V由springboot提供，除非springboot没提供版本号
  2. 如果发生坐标错误，再指定Version（小心版本冲突）

  • 引导类

      @SpringBootApplication //自动扫描包里所有的spring容器
  public class Springboot0101QuickstartApplication {
      public static void main(String[] args) {
          SpringApplication.run(Springboot0101QuickstartApplication.class, args);
      }
  
  }
  1. 引导类是boot工程的入口，执行main方法可以启动项目

    容器：Spring 容器是 Spring 框架的核心，是用来管理对象的。容器将创建对象，把它们连接在一起，配置它们，并管理他们的整个生命周期从创建到销毁。

  2. @SpringBootApplication 自动扫描包里所有的spring容器
  3. springboot工程初始化Spring容器，扫描引导类所在的包加载bean

    bean:简而言之，Spring bean是Spring框架在运行时管理的对象。Spring bean是任何Spring应用程序的基本构建块。你编写的大多数应用程序逻辑代码都将放在Spring bean中。

  • 内嵌tomcat服务器

    tmocat: 本质上也是容器，提供了web服务

补充：REST开发

REST风格

  • 按照rest风格访问资源时使用行为动作区分对资源的操作
  1. GET 查询
  2. POST 新增/保存
  3. PUT 修改/更新
  4. DELETE 删除

  • 根据REST风格访问资源称为RESTFUL

    @RequestBody @RequestParam @PathVariable

  • Param用于接受url地址传参或表单传参

  • Body用于接收json数据

  • Variable用于接收路径参数，使用{参数名称}描述路径参数 —

  • ```java
    @RequestMapping(value = “/users/{id}” ,method = RequestMethod.GET)
    @ResponseBody
    public String getById(@PathVariable Integer id){
    System.out.println(“user getById…”+id);
    return “{‘module’:’user getById’}”;
    }

springboot基础配置

属性配置

  • 修改服务器端口：在applications.properties里面利用键值对修改即可

    配置文件分类

  • properties
  • yml(推荐)
  • yaml
  • 优先级依次递减，相互覆盖叠加

    yaml数据格式

yaml数据读取

  • 在引导类里面用spring的语法格式：@Value(“${待读取的数据}”)

  • 如果数据存在多层级，依次书写层级名称即可

  • 在配置文件中可以使用属性名引用方式引用属性
    baseDir: /user/local/fire

    center:
    dataDir: ${baseDir}/data

  • 属性中出现转义字符要使用双引号包裹

  • 读取全部的yaml属性

          @Autowired
      private Environment env;

  • 读取yaml引用类型属性数据

  1. 自定义对象需要封装的数据

  2. 自定义封装数据的模型类，类中定义字段名称和1中数据名称对应

  3. 在引导类里面引用对象即可
