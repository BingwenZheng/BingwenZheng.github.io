学习资料 ：b站黑马程序员

核心概念

  • Ioc 控制反转

    外部提供对象，程序内部不再new对象

  • spring提供容器，成为Ioc容器，充当外部

    Ioc负责对象的创建初始化等一系列操作，被创建或被管理的对象在Ioc中被称为Bean

  • DI 依赖注入

    在容器中建立bean之间的依赖

    补充知识（dao层和service层）

    Service是业务层，Dao是数据访问层，这样的分层是基于MVC架构来说的，分层的主要作用是解耦。
    对于Spring这样的框架，（View\Web）表示层调用控制层（Controller），控制层调用业务层（Service），业务层调用数据访问层（Dao）。
    具体起来，Dao的作用是封装对数据库的访问：增删改查，不涉及业务逻辑，只是达到按某个条件获得指定数据的要求；
    而Service，则是专注业务逻辑，对于其中需要的数据库操作，都通过Dao去实现；

    Ioc

配置过程：

  1. 导入spring的坐标
  2. bean标签配置bean
  3. 初始化Ioc容器 :

    ApplicationContext ctx = new ClassPathXmlApplicationContext(“配置文件的名称”);

  4. 获取bean

    ctx.getBean(“定义的变量名称”)

DI

  1. 提供依赖对象的setter方法 – 对service里面定义的数据类型定义setter方法

  2. 配置service与dao之间的关系

     <bean id = "bookService" name="xxxxx" class="com.itheima...">// 定义bean容器，id是需要装填的对象 可以用name字段设置别名
     <property name="bookDao" ref="bookDao"/>  //前面的Dao是service里面定义的数据类型，后面的Dao是依赖注入的bean容器
 </bean>

bean的配置

在spring中默认的bean为单例，也就是说通过两次定义bean其容器的地址是一样的

在bean的创建过程中，通过修改scope关键词的属性为“prototype”来变成非单例

  • 适用于单例的对象类型

    bean的实例化

  • 静态工厂实例化bean–(了解)
  • 实例工厂实例bean– 要先创建一个对象，在bean中引用bean（了解）

注解开发

只需要在所需的类上方添加注解，然后再xml文件里面添加扫描的依赖 – component-scan（了解）

Spring提供的三个衍生注解

  • @Controller ： 表现层bean定义
  • @Service : 业务层
  • @Repository ： 数据层

纯注解开发模式

摒弃了原来还需要在xml里面的配置，直接新建Java配置类如下

    @Configuration//设置当前类为配置类
    @ComponentScan("包名")//设置扫描路径，只能添加一次
    public class SpringConfig{}

并且原先注入配置文件的容器初始化方法改为：

    ApplicationContext ctx = new AnnotationConfigApplicationContext(配置类的名称);
